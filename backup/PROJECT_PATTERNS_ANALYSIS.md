# UniLearn Java Project - Architecture & Patterns Analysis

## Overview
This is a JavaFX desktop application (Maven-based) with Hibernate ORM integration for database operations. The project follows a layered architecture with clear separation of concerns across entities, repositories, services, and controllers.

---

## 1. PACKAGE NAMING CONVENTIONS

**Base Package Structure:**
```
entities/          - JPA/Hibernate entity classes
controller/        - JavaFX controllers (FXML-based)
service/          - Business logic services

dto/              - Data Transfer Objects
  └── lms/        - LMS module DTOs
```

## 2. ENTITY PATTERNS (Hibernate/JPA)

### General Characteristics:
- **Generated by:** Hibernate Tools 6.6.42.Final
- **JPA Version:** Jakarta Persistence 3.1.0 (not javax.persistence)
- **ID Generation:** `@GeneratedValue(strategy=IDENTITY)` - Auto-increment
- **Relationships:** Eager/Lazy loading with `FetchType.LAZY` (default)

### Annotation Patterns:

```java
@Entity
@Table(name="user", 
    uniqueConstraints = @UniqueConstraint(columnNames="email")
)
public class User implements java.io.Serializable {
    
    @Id @GeneratedValue(strategy=IDENTITY)
    @Column(name="id", unique=true, nullable=false)
    private Integer id;
    
    @Column(name="title", nullable=false, length=255)
    private String title;
    
    @Temporal(TemporalType.TIMESTAMP)
    @Column(name="created_at", nullable=false, length=19)
    private Timestamp createdAt;
    
    // One-to-Many relationships
    @OneToMany(mappedBy="parentEntity", fetch=FetchType.LAZY)
    private Set<ChildEntity> children = new HashSet<>(0);
    
    // Many-to-One relationships
    @ManyToOne(fetch=FetchType.LAZY)
    @JoinColumn(name="program_id", nullable=false)
    private Program program;
}
```

### Key Features:

1. **Data Types:**
   - `Integer` for primary keys
   - `String` for text content
   - `Timestamp` (java.sql) for datetime fields
   - `Date` (java.sql) for date-only fields
   - `byte` for boolean flags (0/1)
   - `Set<T>` with `HashSet` for collections

2. **Common Patterns:**
   - No-arg constructor `public Entity() {}`
   - Full constructor with all parameters
   - Constructor with mandatory fields only
   - Getter/setter methods for all fields
   - `implements java.io.Serializable` (for serialization)
   - Convention: Use singular naming for many-to-one relationships
   - Convention: Use plural naming for one-to-many collections

3. **Relationship Examples:**

   **One-to-Many (Parent):**
   ```java
   @OneToMany(mappedBy="user")
   private Set<Schedule> schedules = new HashSet<>();
   ```

   **Many-to-One (Child):**
   ```java
   @ManyToOne(fetch=FetchType.LAZY)
   @JoinColumn(name="program_id", nullable=false)
   private Program program;
   ```

   **Join Tables (Many-to-Many implied):**
   - StudentClasse: joins Student (User) with Classe
   - ClasseCourse: joins Classe with Course
   - TeacherClasse: joins Teacher (User) with Classe

## 3. REPOSITORY PATTERNS (DAO Layer)

### General Characteristics:
- **Pattern:** DAO-like classes (not Spring Data JPA)
- **Not Interface-Based:** Direct implementation classes
- **Session Management:** Uses `HibernateSessionFactory.getSession()`
- **Transaction Handling:** Manual transaction management
- **Error Handling:** Try-catch with logging and IllegalStateException

### Repository Structure:

```java
public class UserRepository {
    
    private static final Logger LOGGER = LoggerFactory.getLogger(UserRepository.class);
    private static final String SORT_FIELD_CREATED_AT = "createdAt";
    private static final int DEFAULT_LIMIT = 20;
    
    // CRUD Operations
    public Optional<User> findById(Long id) {
        Session session = HibernateSessionFactory.getSession();
        try {
            return Optional.ofNullable(session.get(User.class, userId));
        } catch (Exception exception) {
            LOGGER.error("Failed to fetch user by id: {}", id, exception);
            throw new IllegalStateException("Unable to fetch user by id", exception);
        } finally {
            HibernateSessionFactory.closeSession();
        }
    }
    
    // List with Pagination
    public List<User> findAll(int offset, int limit, String sortBy, boolean ascending) {
        Session session = HibernateSessionFactory.getSession();
        try {
            CriteriaBuilder builder = session.getCriteriaBuilder();
            CriteriaQuery<User> criteria = builder.createQuery(User.class);
            Root<User> root = criteria.from(User.class);
            
            // Build query with order by
            if (ascending) {
                criteria.orderBy(builder.asc(root.get(safeSortBy)));
            } else {
                criteria.orderBy(builder.desc(root.get(safeSortBy)));
            }
            
            Query<User> query = session.createQuery(criteria);
            query.setFirstResult(Math.max(offset, 0));
            query.setMaxResults(Math.max(limit, 1));
            return query.getResultList();
        } catch (Exception exception) {
            LOGGER.error("Failed to fetch users", exception);
            throw new IllegalStateException("Unable to fetch users", exception);
        } finally {
            HibernateSessionFactory.closeSession();
        }
    }
    
    // Custom Query with HQL
    public Optional<User> findByEmail(String email) {
        Session session = HibernateSessionFactory.getSession();
        try {
            Query<User> query = session.createQuery(
                "from User u where lower(u.email) = :email",
                User.class
            );
            query.setParameter("email", email.trim().toLowerCase());
            return query.uniqueResultOptional();
        } catch (Exception exception) {
            LOGGER.error("Failed to fetch user by email", exception);
            throw new IllegalStateException("Unable to fetch user by email", exception);
        } finally {
            HibernateSessionFactory.closeSession();
        }
    }
    
    // Save/Update
    public User save(User user) {
        Session session = HibernateSessionFactory.getSession();
        Transaction transaction = null;
        try {
            transaction = session.beginTransaction();
            User merged = (User) session.merge(user);
            transaction.commit();
            return merged;
        } catch (Exception exception) {
            rollback(transaction);
            LOGGER.error("Failed to save user", exception);
            throw new IllegalStateException("Unable to save user", exception);
        } finally {
            HibernateSessionFactory.closeSession();
        }
    }
    
    // Delete
    public void delete(User user) {
        Session session = HibernateSessionFactory.getSession();
        Transaction transaction = null;
        try {
            transaction = session.beginTransaction();
            User managed = session.get(User.class, user.getId());
            if (managed != null) {
                session.remove(managed);
            }
            transaction.commit();
        } catch (Exception exception) {
            rollback(transaction);
            LOGGER.error("Failed to delete user", exception);
            throw new IllegalStateException("Unable to delete user", exception);
        } finally {
            HibernateSessionFactory.closeSession();
        }
    }
    
    private void rollback(Transaction transaction) {
        if (transaction != null && transaction.isActive()) {
            transaction.rollback();
        }
    }
}
```

## 4. SERVICE PATTERNS (Business Logic)

### General Characteristics:
- **Not Interface-Based:** Direct implementation classes
- **Naming Convention:** `XxxService` (singular)
- **Method Names:** Start with verb (create, update, get, delete, find, etc.)
- **Session Management:** Manual via HibernateSessionFactory
- **Dependencies:** Services depend on repositories or other services
- **Exception Handling:** Custom validation with IllegalArgumentException, wrapped in RuntimeException

### Service Structure:

```java
public class UserService {
    
    private static final Logger LOGGER = LoggerFactory.getLogger(UserService.class);
    private static final int DEFAULT_PAGE_SIZE = 20;
    private static final int BCRYPT_ROUNDS = 12;
    
    // Conversion methods
    public List<dto.lms.UserOptionDto> getAllUsersOptionsDto() {
        return getAllUsers(1, 1000).stream()
            .map(u -> new dto.lms.UserOptionDto(u.getId(), u.getEmail()))
            .collect(java.util.stream.Collectors.toList());
    }
    
    // Retrieve operations
    public List<User> getAllUsers() {
        return getAllUsers(1, DEFAULT_PAGE_SIZE);
    }
    
    public List<User> getAllUsers(int page, int pageSize) {
        Session session = HibernateSessionFactory.getSession();
        try {
            int safePage = Math.max(page, 1);
            int safePageSize = Math.max(pageSize, 1);
            
            Query<User> query = session.createQuery(
                "from User u order by u.createdAt desc, u.id desc",
                User.class
            );
            query.setFirstResult((safePage - 1) * safePageSize);
            query.setMaxResults(safePageSize);
            return query.getResultList();
        } catch (Exception exception) {
            LOGGER.error("Failed to fetch users with pagination", exception);
            throw new IllegalStateException("Unable to fetch users", exception);
        } finally {
            HibernateSessionFactory.closeSession();
        }
    }
    
    public Optional<User> getUserByEmail(String email) {
        if (email == null || email.isBlank()) {
            return Optional.empty();
        }
        // ... session management and query
    }
    
    // Create operations with validation
    public User createUser(User user, String tempPassword) {
        if (user == null) {
            throw new IllegalArgumentException("User payload is required");
        }
        if (tempPassword == null || tempPassword.isBlank()) {
            throw new IllegalArgumentException("Temporary password is required");
        }
        
        Session session = HibernateSessionFactory.getSession();
        Transaction transaction = null;
        try {
            transaction = session.beginTransaction();
            
            // Business logic here
            user.setPassword(BCrypt.hashpw(tempPassword, BCrypt.gensalt(BCRYPT_ROUNDS)));
            user.setCreatedAt(Timestamp.from(Instant.now()));
            user.setUpdatedAt(Timestamp.from(Instant.now()));
            
            session.persist(user);
            transaction.commit();
            return user;
        } catch (Exception exception) {
            if (transaction != null && transaction.isActive()) {
                transaction.rollback();
            }
            LOGGER.error("Failed to create user", exception);
            throw new IllegalStateException("Unable to create user", exception);
        } finally {
            HibernateSessionFactory.closeSession();
        }
    }
    
    // Helper methods for security
    private String hashPassword(String password) {
        return BCrypt.hashpw(password, BCrypt.gensalt(BCRYPT_ROUNDS));
    }
}
```

## 5. CONTROLLER PATTERNS (JavaFX)

### General Characteristics:
- **Framework:** JavaFX with FXML
- **Pattern:** MVC-like with `Initializable` interface
- **FXML Binding:** @FXML annotation for UI components
- **Event Handling:** Handler methods with @FXML prefix
- **Service Injection:** Services instantiated as private final fields

### Controller Structure:

```java
public class UserFormController implements Initializable {
    
    // Constants
    private static final Pattern EMAIL_PATTERN = Pattern.compile("^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+$");
    private static final Pattern PHONE_PATTERN = Pattern.compile("^[+0-9()\\-\\s]{7,20}$");
    
    // FXML injected components
    @FXML
    private TextField emailField;
    @FXML
    private ComboBox<String> roleComboBox;
    @FXML
    private TextArea descriptionArea;
    @FXML
    private CheckBox sendWelcomeEmailCheckBox;
    
    // Service instances
    private final UserService userService = new UserService();
    private final ProfileService profileService = new ProfileService();
    private final EmailService emailService = new EmailService();
    
    // State variables
    private boolean editMode;
    private User editingUser;
    private Runnable onSaveSuccess;
    
    @Override
    public void initialize(URL location, ResourceBundle resources) {
        // UI initialization
        roleComboBox.setItems(FXCollections.observableArrayList("ADMIN", "TEACHER", "STUDENT"));
        statusComboBox.setItems(FXCollections.observableArrayList("Active", "Inactive"));
        setCreateMode();
    }
    
    // Mode switching
    public void setCreateMode() {
        editMode = false;
        editingUser = null;
        clearForm();
        emailField.setDisable(false);
        generateTempPassword();
    }
    
    public void setEditMode(User user) {
        editMode = true;
        editingUser = user;
        populateForm(user);
        emailField.setDisable(true);
    }
    
    // Event handlers
    @FXML
    private void handleSave() {
        if (!validateForm()) {
            return;
        }
        
        try {
            User user = buildUserFromForm();
            if (editMode) {
                userService.updateUser(user);
            } else {
                userService.createUser(user, generatedTempPassword);
            }
            
            if (onSaveSuccess != null) {
                onSaveSuccess.run();
            }
            showSuccess("User saved successfully");
        } catch (Exception exception) {
            showError(safeErrorMessage(exception));
        }
    }
    
    @FXML
    private void handleCancel() {
        closeDialog();
    }
    
    // Form management
    private boolean validateForm() {
        String email = emailField.getText();
        if (email == null || !EMAIL_PATTERN.matcher(email).matches()) {
            showError("Invalid email format");
            return false;
        }
        return true;
    }
    
    private User buildUserFromForm() {
        User user = editMode ? editingUser : new User();
        user.setEmail(emailField.getText().toLowerCase().trim());
        user.setRole(roleComboBox.getSelectionModel().getSelectedItem());
        return user;
    }
    
    private void populateForm(User user) {
        emailField.setText(user.getEmail());
        roleComboBox.getSelectionModel().select(user.getRole());
    }
    
    // Callbacks
    public void setOnSaveSuccess(Runnable onSaveSuccess) {
        this.onSaveSuccess = onSaveSuccess;
    }
    
    // Dialog management
    private void showError(String message) {
        Alert alert = new Alert(Alert.AlertType.ERROR);
        alert.setTitle("Error");
        alert.setContentText(message);
        alert.showAndWait();
    }
    
    private void closeDialog() {
        Stage stage = (Stage) emailField.getScene().getWindow();
        stage.close();
    }
}
```

```java
    }

            return;

    }

        try {
            FXMLLoader loader = new FXMLLoader(
                getClass().getResource("/view/user/UserForm.fxml")
            );

            UserFormController controller = loader.getController();
            
            if (user != null) {
                controller.setEditMode(user);
            }
            controller.setOnSaveSuccess(() -> refreshUsers());
            
            Stage stage = new Stage();
            stage.setScene(new Scene(root));
            stage.initModality(Modality.WINDOW_MODAL);
            stage.showAndWait();
        } catch (IOException exception) {
            LOGGER.error("Failed to load user form", exception);
            showError("Unable to open user form");
        }
    }
    
    private void refreshUsers() {
        List<User> users = userService.getAllUsers(currentPage, PAGE_SIZE);
        filteredUsers.clear();
        filteredUsers.addAll(users);
    }
}
```

## 6. DTO PATTERNS

### General Characteristics:
- **Immutable:** Final fields, no setters
- **Constructor-Based:** Constructor takes all parameters
- **Simple Structure:** Minimal logic, mainly getters
- **toString() Override:** For display in ComboBox
- **Package:** `dto.lms` or `dto.job_offer` by module

## 7. SECURITY & SESSION MANAGEMENT

### UserSession Pattern:

```java
package security;

public final class UserSession {
    
    private static volatile Integer currentUserId;
    private static volatile String currentUserRole;
    
    private UserSession() { }
    
    public static void setCurrentUser(User user) {
        if (user == null) {
            currentUserId = null;
            currentUserRole = null;
        } else {
            currentUserId = user.getId();
            currentUserRole = normalizeRole(user.getRole());
        }
    }
    
    public static Optional<Integer> getCurrentUserId() {
        return Optional.ofNullable(currentUserId);
    }
    
    public static String getCurrentUserRole() {
        return currentUserRole != null ? currentUserRole : "";
    }
    
    public static boolean isLoggedIn() {
        return currentUserId != null;
    }
    
    public static void clear() {
        currentUserId = null;
        currentUserRole = null;
    }
    
    private static String normalizeRole(String role) {
        if (role == null || role.isBlank()) return "";
        String r = role.trim().toUpperCase();
        return r.startsWith("ROLE_") ? r.substring(5) : r;
    }
}
```

### Key Features:
- **Thread-Safe:** Uses `volatile` keyword
- **Thread-Local Pattern:** Per-request/per-session storage
- **Role Normalization:** Handles both "ROLE_ADMIN" and "ADMIN" formats
- **Optional Usage:** Returns `Optional` for userId
- **Module:** [src/main/java/security/UserSession.java](src/main/java/security/UserSession.java)

## 8. HIBERNATESESSIONFACTORY PATTERN

### Session Management:

```java
public final class HibernateSessionFactory {
    
    private static final Logger LOGGER = LoggerFactory.getLogger(HibernateSessionFactory.class);
    private static final String ENTITY_PACKAGE = "entities";
    private static SessionFactory sessionFactory;
    private static final ThreadLocal<Session> CURRENT_SESSION = new ThreadLocal<>();
    
    private HibernateSessionFactory() { }
    
    public static synchronized SessionFactory getInstance() {
        if (sessionFactory == null || sessionFactory.isClosed()) {
            sessionFactory = buildSessionFactory();
        }
        return sessionFactory;
    }
    
    public static Session getSession() {
        Session session = CURRENT_SESSION.get();
        
        if (session == null || !session.isOpen()) {
            try {
                session = getInstance().openSession();
                CURRENT_SESSION.set(session);
            } catch (HibernateException exception) {
                LOGGER.error("Unable to open Hibernate session", exception);
                throw new IllegalStateException("Database connection failed", exception);
            }
        }
        
        return session;
    }
    
    public static void closeSession() {
        Session session = CURRENT_SESSION.get();
        if (session != null) {
            // close logic
        }
    }
}
```

### Key Patterns:
- **Singleton Pattern:** Static synchronized getInstance()
- **Thread-Local Sessions:** Per-thread session caching
- **Auto-Configuration:** Scans entity package automatically

## 10. CROSS-CUTTING CONCERNS

### Exception Handling Strategy:
1. **Validation Exception:** `IllegalArgumentException` at service layer
2. **Data Access Exception:** Catch, log, then throw `IllegalStateException`
3. **UI Exception:** Caught in controller, shown to user via Alert dialog

### Logging Strategy:
- **SLF4J** with Logback backend
- **Logger Pattern:** `private static final Logger LOGGER = LoggerFactory.getLogger(ClassName.class);`
- **Levels:** 
  - ERROR: Failed operations with exception
  - WARN: Unexpected but handled situations
  - INFO: General operation flow (less common)
  - DEBUG: Development-level details

### Password Management:
- **Hashing:** jBcrypt with 12 rounds of salting
- **Verification:** Direct comparison using BCrypt.checkpw()
- **Storage:** Hashed only, plain text never stored

### Timestamp Handling:
- **Type:** `java.sql.Timestamp` (not java.util.Date)
- **Creation:** `Timestamp.from(Instant.now())`
- **Annotation:** `@Temporal(TemporalType.TIMESTAMP)` on entity fields

## 12. DEVELOPMENT GUIDELINES

### When Creating New Entities:
1. Follow the generated pattern with all three constructors
2. Use `Integer` for ID (IDENTITY generation)
3. Use `Timestamp` for datetime fields
4. Use `Set<T>` with `HashSet<>` for collections
5. Mark relationships with LAZY loading by default
6. Add unique constraints at table level if needed

### When Creating New Repositories:
1. Create method names: `findAll()`, `findById()`, `findByXxx()`, `save()`, `delete()`
2. Always use try-catch-finally with session management
3. Always log errors with context
4. Always validate input parameters
5. Support pagination for list operations
6. Return `Optional<T>` for single-result queries

### When Creating New Services:
1. Inject repository dependencies if needed
2. Validate all inputs, throw `IllegalArgumentException` for bad data
3. Handle transactions manually
4. Log operations at entry points and in catch blocks
5. Convert entities to DTOs when needed
6. Support pagination in list methods

### When Creating New Controllers:
1. Implement `Initializable`
2. Inject all services as private final fields
3. Configure UI components in `initialize()`
4. Use @FXML handlers for button clicks
5. Validate form data before service calls
6. Show user-friendly error messages
7. Support both Create and Edit modes for forms
8. Use callbacks (`setOnXxxSuccess()`) for parent-child communication

### DTOs:
1. Make all fields final and private
2. Set values only via constructor
3. Provide appropriate getters
4. Override `toString()` for display purposes
5. Keep DTO logic minimal (no business logic)
